Introducere

1.1 Cerinta

Pe baza unui API REST ori GraphQL propriu, realizati o aplicatie Web – disponibila, de asemenea, ca extensie de navigator Web – care recomanda utilizatorilor autentificati sau nu diverse dispozitive electronice (telefoane, tablete, ceasuri, drone,…) in functie de pret ori alte caracteristici – e.g., model similar, facilitati, culoare, autonomie etc.
Datele analizate vor fi preluate din surse multiple (fluxuri de stiri, API-uri disponibile, via scraping) oferite de situri de profil ce vor putea fi precizate de utilizator. Statisticile celor mai populare produse si recomandarile generate vor fi disponibile si sub forma de fluxuri de stiri RSS si partajate pe minim o retea sociala folosind hashtag-ul #redevice. Se vor avea in vedere aspecte precum internationalizarea si accesibilitatea.

1.2 Publicul tinta

Acest document este pentru consumatori acestui produs care sunt interesati de detaliile tehnice si cum poate fi utilizata aplicatia pentru a indeplini cerinta clientului.

1.3 Scopul Produsului

Produsul isi doreste sa-si ajute utilizatori prin intermediul unor recomandari ale produselor de pe site-urile cerute de catre utilizator astfel sa ii fie mai facil in a gasi produse care sunt considerate bune de catre comunitate.

Descriere generala

2.1 Functiile Produsului

Utilizatorul poate cauta produse, poate sa le sorteze dupa preferintele lui, poate sa vada proprietatile fiecarui produs, sa dea share pe twitter, sa se autentifice.

2.1.1 Contul de admin

Conturile de administrator sunt introduse de catre admin. Exista o pagina speciala pentru a administra conturile si produsele. El poate sa le stearga.

2.1.2 Share pe o platforma media

Admin-ul poate da share la topul celor mai bune produse si o sa fie partajate pe o platforma twitter. Link-ul la contul respectiv este aici Se vor genera top 10 cele mai bune produse din Romania.

2.1.3 Extensie de navigator web

Aplicatia este disponibila ca si extensie de navigator web. Se va apela api-ul de produse si se vor afisa produsele.

2.2 Clasele de utilizatori

Utilizatori vor fi oameni pasionati de produse it care nu sunt siguri de ce decizie sa faca si au nevoie de un mic ajutor.

2.3 Sistem de operare

Aplicatia va rula in orice sistem de operare Windows avand nevoie access la baza de date si acces la internet.

2.4 Documentatie utilizator

In cadrul proiectului se gaseste un manual de utilziare pentru a il ajuta pe consumator in intelegerea site-ului.

2.4.1 Documentatie OpenApi

Pe langa documentatie pentru utilizator, este si documentatie OpenAPI care descrie despre api-urile aplicatiei.

Interfate

3.1 Interfata Utilizator

3.1.1 Homepage (index) (Brezuleanu Alexandru)

Pagina HomePage(index) este structurata in patru sectiuni. O prima sectiune cuprinde logo-ul firmei, bara de cautarea precum si butoanele de login, respectiv sign up, fiind urmata de a doua sectiune ce cuprinde bara de meniu prin intermediul careia pot fi accesate categorii de produse si informatii despre firma si noutati. A treia sectiune este divizata in doua: filtrul de cautare, prin intermediul caruia vor putea fi filtrate gadget-urile in functie de preferinta utilizatorului, respectiv lista de produse ce va contine itemele sugerate clientului. Ultima sectiune contine link-uri utile ce pot simplifica interactiunea utilizatorului cu aplicatia.
Butoanele de login si sign up duc spre pagina de logare, respectiv cea de inregistrare, in timp ce bara de cautare va permite afisarea doar acelor produse care respecta un pattern specificat de utilizator. Sectiunea care cuprinde bara de meniu contine link-uri catre paginile respective, iar in cazul unor rezolutii mai mici dispare, fiind inlocuita de catre un buton care o face sa apara intr-un mod convenabil rezolutiei. Filtrul de cautare contine mai multe checkbox-uri ce permit alegerea unor produse care sa intalneasca doar acele specificatii(coloane din tabela products a bazei de date) precizate de client. Lista cu produse va contine cate un chenar specific fiecarui gadget, in care se va putea specifica ratingul dat de utilizator dar care va permite si plasarea acelui item in categoria favorite. La rezolutii mai mici, ambele sectiuni schimba modul de afisare a informatiilor din cel orizontal in cel vertical, fiind astfel mai usor de accesat si vizualizat informatia. Ultima sectiunea cuprinde link-uri catre pagini utile utilizatorului sau catre informatii referitoare la aplicatie/membrii echipei, fiind menita astfel sa sporeasca aplicabilitatea site-ului.

3.1.1.a) Procesul de afisarea a produselor (Brezuleanu Alexandru)

Sectiunea ce cuprinde filtrele impreuna cu searchbar-ul sunt incluse intr-un form iar in momentul in care utilizatorul si-a setat criteriiile pe baza caruia doreste sa filtreze produsele, apasa butonul FILTER care seteaza filtrele selectate intr-un $_GET array.
Se va apela mai apoi un API in functie de variabile trimise care va returna lista de produse cautate in format JSON de unde vor fi mai apoi convertite in obiecte de tip Product si afisate utilizatorului.
API-ul de generare a listei de produse este: http://localhost/PHP/Controller/ProductController.php?Method=getFiltredProducts

3.1.2 Item view (Baranceanu Vlad)

Sectiunea unde este prezentat produsul contine imagini ale produsului si informatii despre pret si specificatii, acestea fiind preluate din baza de date. 3.1.3 Procesul de logare (Opariuc Rares Ioan)

Pagina de Login este structurata astfel: animatia cu logo-ul firmei care se afla pe prima pagina si procesul de logare al utilizatorului.
Cand utilizatorul se inregistreaza, apare in stanga un gif cu logo-ul firmei
Procesul de logare al utilizatorului se foloseste de un form si de butoane auxiliare. Cand este apasat butonul de home, utilizatorul este redirectionat la pagina principala. Cand utilizatorul introduce Username-ul si Parola, se va verifica in baza de date daca exista utilizatorul respectiv in maniera AJAX. Se va apela un API de tip POST si in functie de variabile trimise, se va genera un mesaj JSON. In functie de limbajul de limbajul sesiunii pe care a ales-o utilizatorul,se va afisa mesajul corespunzator. API-ul de logare este http://localhost/PHP/Controller/loginController.php In caz afirmativ, utilizatorul va fi redirectionat la pagina principala si va fi conectat. In caz negativ, pagina web il va atentiona ca nu exista utilizatorul respectiv, respectiv nu este corecta parola.
Butonul de Remember me va retine utilizatorul cand va redeschide pagina si astfel acesta va fi conectat automat. Se va folosi un cookie pentru a retine datele respective. In cookie, se va retine un token alcatuit dintr-un selector si validator hashat ce va fi salvat in baza de date user_token. Cand utilizatorul va redeschide pagina si a bifat optiunea de cookie, se va verifica daca cookie-ul e expirat sau selectorul este schimbat. Daca da, se va sterge cookie-ul respectiv, iar utilizatorul va fi rugat sa se logheze din nou. Daca utilizatorul vrea sa isi faca un cont nou, va apasa butonul de sign up care il va redirectiona la pagina de sign up.
In cazul in care utilizatorul este pe telefon, va disparea animatia si va ramane procesul de logare (rezolutia maxima este de 760 px).
Daca utilizatorul vrea sa isi creeze un cont nou, va apasa butonul de Sign Up si va fi redirectionat la pagina de sign up. Aceasta va avea aceeasi structura ca pagina de login (animatia si procesul de logare). Utilizatorul trebuie sa introduca username-ul, email-ul, parola noua si sa testeze din nou parola noua. Remember me va avea aceeasi functionalitate ca pagina de login. Pagina de Signup are acelasi Responsive Design ca Login-ul.
Datele vor fi validate la intrarea Signup-ului (email-ul trebuie sa respecte formatul, parola tastata de doua ori sa fie aceeasi). Cand datele au fost introduse corespunzator si se va apasa pe register, se va verifica baza de date pentru a vedea daca exista utilizatori cu acelasi email sau username. In caz negativ, site-ul il va atentiona pe utilizator ca datele sunt deja folosite si trebuie sa le schimbe. In caz afirmativ, vor fi introduse in baza de date informatiile iar utilizatorul va fi redirectionat spre pagina principala in timp ce este logat cu contul respectiv.
In ambele cazuri, daca logarea s-a realizat cu succes, va aparea in dreapta sus numele utilizatorului in timp ce acesta va naviga restul paginilor. Datele vor fi criptate prin intermediul unor functii deja predifinite ( md5(), sha1() ) si vor fi stocate in baza de date pentru securitatea informatiilor.

3.1.4 Procesul de inregistrare (Opariuc Rares Ioan)

Pagina de Login este structurata astfel: animatia cu logo-ul firmei care se afla pe prima pagina si procesul de logare al utilizatorului.
Cand utilizatorul se inregistreaza, apare in stanga un gif cu logo-ul firmei
Procesul de inregistrare al utilizatorului se foloseste de un form si de butoane auxiliare. Cand este apasat butonul de home, utilizatorul este redirectionat la pagina principala. Cand utilizatorul introduce datele necesare, se va verifica in baza de date daca exista username-ul si email-ul exista in baza de date respectiv in maniera AJAX. Se va apela un API de tip POST si in functie de variabile trimise, se va genera un mesaj JSON. In functie de limbajul de limbajul sesiunii pe care a ales-o utilizatorul,se va afisa mesajul corespunzator. API-ul de inregistrare este http://localhost/PHP/Controller/signUpController.php In caz afirmativ, utilizatorul va fi redirectionat la pagina de login si se poate conecta cu contul respectiv. In caz negativ, pagina web il va atentiona ca nu a creat contul.

In cazul in care utilizatorul este pe telefon, va disparea animatia si va ramane procesul de logare (rezolutia maxima este de 760 px).

3.1.4 Procesul de schimbare a parolei (Opariuc Rares Ioan)

Procesul de schimbare a parolei al utilizatorului se foloseste de un form si de butoane auxiliare. Cand este apasat butonul de home, utilizatorul este redirectionat la pagina principala. Cand utilizatorul introduce datele necesare, se va verifica in baza de date daca exista username-ul, id-ul printr-o variabila de sesiune si daca parola veche si noua a introdus-o bine in maniera AJAX. Se va apela un API de tip POST si in functie de variabile trimise, se va genera un mesaj JSON. In functie de limbajul de limbajul sesiunii pe care a ales-o utilizatorul,se va afisa mesajul corespunzator. API-ul de inregistrare este http://localhost/PHP/Controller/changePasswordController.php In caz afirmativ, utilizatorul si-a schimbat parola respectiva. In caz negativ, pagina web il va atentiona.

In cazul in care utilizatorul este pe telefon, va ramane procesul de schimbare a parolei (rezolutia maxima este de 760 px).
